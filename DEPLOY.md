# アプリの公開方法とプライバシーについて

## 1. 一般公開について
このアプリは「静的Webサイト（HTML/CSS/JSのみ）」で構成されているため、無料で簡単に公開できます。以下のサービスがおすすめです。

### 推奨: Vercel / Netlify / GitHub Pages
これらは「静的サイトホスティング」と呼ばれ、このフォルダの内容をそのままアップロードするだけで世界中に公開できます。サーバー代も基本的にかかりません。

**公開の手順例 (GitHub Pagesの場合):**
1. GitHubにリポジトリを作成し、このフォルダのファイルをアップロードします。
2. リポジトリの「Settings」→「Pages」から、`main` ブランチを選択して保存します。
3. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>` で誰でもアクセスできるようになります。

**公開の手順例 (Vercel/Netlifyの場合):**
1. フォルダをドラッグ＆ドロップするだけで公開できる機能があります。

## 2. プライバシー上の安全性（重要）
結論から言うと、このアプリは**プライバシーの観点から非常に安全**であり、むしろ「安心安全」を売りにできる設計になっています。

### なぜ安全なのか？
通常のWebアプリは、画像をサーバーに送信して加工し、結果送り返します。この場合、「サーバーに画像が保存されるかもしれない」というリスクがあります。

一方、この「PrivacyCam」は、**クライアントサイド処理**という技術を使っています。
- **仕組み**: AI（face-api.js）も画像処理も、すべて**ユーザーのブラウザ（スマホやPC）の中だけ**で動いています。
- **通信なし**: 画像データがインターネットを通じて外部のサーバーに送信されることは**一切ありません**。ネットを切った状態でも（一度読み込めば）動作します。

### 公開時のアドバイス
ユーザーに安心してもらうため、サイト上の目立つ場所に以下の文言を加えることをおすすめします。

> **「安心のプライバシー設計：画像はお客様の端末内（ブラウザ）でのみ処理されます。外部サーバーに画像が送信・保存されることは一切ありません。」**

これにより、ユーザーは自分の子供の写真やプライベートな写真も安心して加工できます。
